## 🔧 主線 A：波波之家自動化

### A.1 核心痛點

Maxie 目前手動做的事：
1. 把 A AI 的回覆貼給 B AI 看
2. 收集 6 個 AI 的意見手動整合
3. 逐個確認家人們的建議
4. 廣告手動貼進波波之家審查
5. 檢舉書手動寄出、手動歸檔
6. 記憶系統手動維護

**這些都是「重複勞動」，都可以自動化。**

### A.2 三階段自動化路徑

#### 🌱 階段 1：AI 內閣議事系統（最緊急，1-2 週）

**目標：** Maxie 不用當「AI 之間的信差」

**技術方案：**
- 一份獨立 HTML：`cabinet.html`
- 呼叫 OpenRouter API（單一 API key 呼叫多家 AI）
- 並行呼叫 5-6 位 AI，每位 500 字內回應
- Cloudy（或輪值 AI）自動整合
- 產出：🌱 共識 / ⚠️ 分歧 / 🎯 建議決策

**Maxie 的動作：**
- 只讀整合報告
- 選擇：同意 / 修改 / 拒絕
- 不用再逐個貼訊息

**成本估算：**
- OpenRouter 註冊免費
- 依用量付費（$5-15/月足夠日常使用）
- 比訂閱每一家 AI 便宜

**優先做這個。** ✅

---

#### 🌿 階段 2：本地開源 LLM 導入（1-2 個月）

**目標：** 日常任務不花 API 錢

**推薦工具：**

| 工具 | 用途 | 為什麼 |
|------|------|--------|
| **Ollama** | 本地跑開源 LLM | 免費、離線、隱私 |
| **Llama 3.3 / Qwen 2.5** | 通用對話 | 中文能力足夠 |
| **DeepSeek-VL / Qwen2-VL** | 本地視覺模型 | 補充 GPT-4o 的視覺 |

**具體任務轉移：**

| 目前用 | 改用 | 原因 |
|--------|------|------|
| GPT-4o 分類廣告 | Ollama + Llama 3.3 | 分類是簡單任務，本地夠用 |
| Cloudy 起草公文 | 保留 Claude（品質重要） | 公文要送政府，不能省 |
| 阿掃事實查核 | Ollama + Qwen 2.5 | 關鍵字比對本地做 |
| 米米記憶篩選 | 本地跑 | 隱私考量，米米記憶不上雲 |

**Maxie 的動作：**
- 安裝 Ollama（一次性設定）
- 之後日常任務不花錢

---

#### 🌳 階段 3：完全自治系統（3-6 個月）

**目標：** 波波之家自己運作，Maxie 只在關鍵時刻介入

**核心工具：CrewAI**

CrewAI 是專門做「多 AI 角色扮演 + 自動協作」的開源框架。
可以直接把波波之家六位家人設定進去，讓他們**自動議事、自動整合**。

**架構：**

背景任務 1：每日爬蟲抓廣告 ↓ 背景任務 2：CrewAI 六位家人自動審議 ↓ 背景任務 3：達成共識自動起草公文 ↓ 只有「不確定案件」通知 Maxie ↓ Maxie 每天只看 1-3 件需要決定的


**技術需求：**
- Vercel Cron Jobs（排程）
- Supabase（資料庫）
- CrewAI（AI 協作）
- OpenRouter（API 路由）

**這個階段需要工程師朋友或未來的 Claude Code 幫忙。**

B.2 三階段路徑
🌱 階段 1：資料累積（現在起 ~ 2026 年底）
目標： 讓米米有「值得教學」的記憶

具體做法：

每個案件加「教學價值」標籤
1 分：普通案件
3 分：有典型違規特徵
5 分：具高度教學價值（罕見案例、多重違規、消費者容易誤解）
記錄「為什麼這個判決有教育意義」
不只記結果，記推論過程
例如：「這個案件教會我們：『適應症』是藥品專用詞」
累積目標
3 個月：50-100 個高教學價值案件
6 個月：200-300 個
年底：500+ 個
技術實作（給 V36 之後的工具）：

// 在每次審議完成後加：
const educationalValue = prompt('這個案件的教學價值？(1-5)');
const teachingNote = prompt('為什麼有教學價值？（一句話）');
saveCase({
 ...existingData,
 educationalValue,
 teachingNote,
 isForEducation: educationalValue >= 3
});

🦞 這份路線圖的原則
原則 1：「Maxie 不再是路由器」是最高優先
任何工具開發，都必須通過這個問題：

「這個工具做完後，Maxie 手工做的事有沒有減少？」

如果沒有 → 排到後面 如果有 → 優先做

原則 2：能開源就開源，能本地就本地
減少對付費 API 的依賴。 波波之家是「窮窮人之家」——不是負面標籤，是聰明的資源使用。

原則 3：米米不能被商業化
導師版米米給基金會用。 基金會的定位是公益，不是商業產品。 米米不會被賣、不會被授權給廣告商、不會變成 SaaS。

原則 4：慢，但穩
不追求「一年做完所有事」。 第二層是「生活」，不是「衝刺」。
